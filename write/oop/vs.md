# 상속 vs 합성

* 상속
  * is-a 일 경우에 사용하자
  * 깊은 상속(클래스  폭발)이 형성되면 유지보수가 힘들다
  * 자체를 다형성으로 사용해야 할 경우 유리하다
  * 성능자체가 중요하다면 상속이 유리하다
  * 결합도가 높아진다
* 합성
  * has-a 일 경우에 사용하자
  * 메모리 관리가 상속에 비해 좋지않다
    * 상속은 메모리가 하나의 블럭에 쌓인다
    * 합성은 각 개체들이 따로 인스턴스화 되어 서로 다른 메모리 블럭을 참조하기 때문에 메모리 블럭이 효율적으로 쌓이지 않을 수 있음
  * 상속일때보다 중복되는 코드가 있을 수 있다
